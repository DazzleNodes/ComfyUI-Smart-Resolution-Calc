# Human Test Checklist: Mask Input Nub

**Target version:** v0.12.0
**Ships with commit:** feat: Mask input for image cutout composite (TBD)
**Companion automated tests:** _to be added_ (this checklist written alongside feature; regression tests in `tests/unit/` and `tests/e2e/` recommended as follow-up)

## What this is

End-to-end human verification for the new optional `mask` input on `SmartResolutionCalculator`. When a mask is connected alongside an image, the node cuts out parts of the input image per the mask and fills the cut-out region with either the connected `fill_image` (if present) or the configured `fill_type` pattern. The mask is auto-fit to the final output dimensions regardless of `output_image_mode`, so this is expected to "just work" across all existing modes.

This checklist is run by a human in ComfyUI with the actual rendering pipeline; mocks cannot verify visual correctness, device-alignment under GPU noise fills, ComfyUI cache invalidation across the JS/Python boundary, or how the cut composite interacts with downstream `VAEDecode → Preview`.

## Prerequisites

1. ComfyUI installed at `C:\code\ComfyUI_experiment` (or equivalent) with SDXL VAE available.
2. Dev junction in place: `C:\code\ComfyUI_experiment\custom_nodes\smart-resolution-calc` → `C:\code\smart-resolution-calc-repo\github`.
3. The `smart-resolution-calc` entry inside `comfyui-dazzlenodes` is in `disabled` state (`python scripts/dev_mode.py status` shows it).
4. Optional but recommended: `DazzleNodes/dazzle-comfy-plasma-fast` installed so `DazNoise:*` fill types are available (device-alignment testing requires a CUDA-bound fill).
5. A test image with a companion mask — the shipped workflow `docs/workflow/SmartResCalc-Mask-Test-Workflow.json` expects `test-920x1022_clipspace-masked.png` (clipspace-drawn mask). Any LoadImage with a clipspace mask works.

**State isolation:** none needed — ComfyUI workflows are self-contained JSON. Just restart ComfyUI after code changes.

**Restart semantics:**
- Code changes to `py/image_utils.py` → browser refresh (hard reload).
- Code changes to `py/smart_resolution_calc.py` including `IS_CHANGED` or class-level attributes → **full ComfyUI server restart**, not just refresh.

## High-Value Verification (~5 minutes)

If you only have 5 minutes, run these. They cover the user-visible "did the work ship."

### HV.1 — Mask nub appears in the correct position

Validates: INPUT_TYPES ordering.

- [ ] **Steps:**
  1. Add a fresh `SmartResolutionCalc` node.
  2. Inspect the left-side input sockets, top-to-bottom.
- [ ] **Expected:** `mask` appears **above** `dazzle_options`, which is above `dazzle_signal`. The nub is labeled `mask` and accepts `MASK` type.
- [ ] **Why this matters:** The ask specified this exact position. If it's missing or in the wrong spot, the UX contract failed.

### HV.2 — Basic cutout (img2img)

Validates: end-to-end compositing, `use_image_for_output=True` gate.

- [ ] **Steps:** Load `docs/workflow/SmartResCalc-Mask-Test-Workflow.json`. Set `image_purpose = img2img`, `fill_type = random`, `output_image_mode = auto`. Queue.
- [ ] **Expected:** The left PreviewImage (direct `image` output) shows the input image with the masked region replaced by random noise. The outline of the cut matches the mask shape.
- [ ] **Check terminal:** `[SmartResCalc] Mask gate: mask=yes, use_image_for_output=True, actual_mode='transform (distort)', image=yes` followed by `[SmartResCalc] Mask composite APPLIED (mean mask=...)`.
- [ ] **Why this matters:** This is the happy-path. If this fails, nothing else works.

### HV.3 — DazNoise fill on GPU (device alignment)

Validates: `composite_with_mask` moves `bg` and `mask` to `fg.device`.

- [ ] **Steps:** With same workflow, switch `fill_type` to `DazNoise:pink`. Queue a few times. Also try `DazNoise:plasma`, `DazNoise:brown`, `DazNoise:greyscale`, `DazNoise:gaussian`.
- [ ] **Expected:** Each run produces a valid composite image. No error in terminal.
- [ ] **Check terminal:** No `ERROR: Mask composite failed: Expected all tensors to be on the same device` lines. `Mask composite APPLIED` appears for each run.
- [ ] **Why this matters:** Regression guard for the cuda/cpu mismatch we shipped in this release.

### HV.4 — Mask edit invalidates cache (IS_CHANGED)

Validates: `IS_CHANGED` fingerprints the mask via shape + sum + mean.

- [ ] **Steps:**
  1. Turn `fill_seed.on = false` (or any fixed seed) so that seed doesn't change across runs.
  2. Queue once with mask A. Note the output.
  3. Right-click the LoadImage, open in MaskEditor, paint a different mask area. Save.
  4. Queue again without touching anything else.
- [ ] **Expected:** The node re-executes (you see a fresh `[SmartResCalc] Mask gate:` line and a new composite), and the output reflects the new mask.
- [ ] **Why this matters:** Before the IS_CHANGED fix, ComfyUI would serve the stale cached output unless the seed happened to roll.

### HV.5 — No mask = no regression

Validates: `mask=None` path is bitwise identical to pre-v0.12.0 behavior.

- [ ] **Steps:** Disconnect the mask from the node. Queue with each `image_purpose` mode: `img2img`, `dimensions only`, `img2noise`, `image + noise`, `img2img + img2noise`.
- [ ] **Expected:** Each mode behaves exactly as in v0.11.4 — no composite, no `Mask: cut` in info string, no unexpected error.
- [ ] **Check terminal:** `[SmartResCalc] Mask gate: mask=no, ...` for each run. No `Mask composite APPLIED` line.
- [ ] **Why this matters:** The feature is additive; existing workflows must keep working.

### HV.6 — Spectral shape uses the cut (img2noise)

Validates: `_generate_latent` Path 3 applies mask to its pattern source.

- [ ] **Steps:** `image_purpose = img2noise`, `blend_strength = 1.0`, `cutoff = 0.2`. Connect LATENT → VAE Decode → Preview. Queue with mask connected.
- [ ] **Expected:** The VAE-decoded preview of the latent shows a noise field whose structure is shaped by the cut region, not by the full input image (compare with mask disconnected — difference should be obvious when mask covers a small area).
- [ ] **Check terminal:** `[SmartResCalc] img2noise pattern: mask composite APPLIED`.
- [ ] **Why this matters:** This is the fix for "the spectral injection showed the entire button instead of just the test mask."

After HV.1–HV.6 pass, mask input is shipping correctly. Detailed sections below cover corner cases.

## Section 1: Output mode × mask interaction (~5 min)

Run each with `image_purpose=img2img`, `fill_type=random`, mask connected.

### 1.1 `output_image_mode = "auto"` (→ transform (distort))

- [ ] **Steps:** Queue with default `auto` setting.
- [ ] **Expected:** Composite shown at final calculated dims; cut shape maps 1:1 onto the distorted image (mask auto-fits to target).

### 1.2 `output_image_mode = "transform (distort)"`

- [ ] **Expected:** Same as 1.1 explicitly.

### 1.3 `output_image_mode = "transform (crop/pad)"`

- [ ] **Steps:** Queue.
- [ ] **Expected:** Composite at target dims; mask fits the crop/pad result, cut region corresponds to the visible image portion.

### 1.4 `output_image_mode = "transform (scale/crop)"`

- [ ] **Expected:** Image scaled to cover, excess cropped. Mask fits the cropped result.

### 1.5 `output_image_mode = "transform (scale/pad)"`

- [ ] **Expected:** Image scaled to fit, padded remainder. Mask fits the padded canvas; cut of the padded area replaced by fill.

### 1.6 `output_image_mode = "empty"`

- [ ] **Steps:** Queue.
- [ ] **Expected:** Full fill-pattern image (no input image pixels). Mask is ignored — no composite, info string has no `Mask: cut`.
- [ ] **Check terminal:** `actual_mode='empty'` and gate not passed.

---

## Section 2: image_purpose × mask interaction (~5 min)

Run each with `output_image_mode=auto`, `fill_type=random`, mask connected.

### 2.1 `img2img`

- [ ] **Expected:** IMAGE = composite. LATENT = VAE-encoded composite. Downstream VAE-decode should reproduce the composite.

### 2.2 `dimensions only`

- [ ] **Expected:** IMAGE = fill pattern (no image pixels). Mask ignored silently. `[SmartResCalc] Mask gate:` shows `use_image_for_output=False`, no `APPLIED` line.

### 2.3 `img2noise`

- [ ] **Expected:** IMAGE = fill pattern. LATENT = image-shaped noise, shape derived from the **cut** image (verify via VAE-decode of latent — HV.6 covers the quick check; here also verify small mask = small structure contribution).

### 2.4 `image + noise`

- [ ] **Expected:** IMAGE = composite (cut image). LATENT = raw noise latent (independent of image). KSampler downstream would generate txt2img-style noise starting point.

### 2.5 `img2img + img2noise`

- [ ] **Expected:** IMAGE = composite. LATENT = VAE-encoded composite + image-shaped noise derived from the composite (phase-randomized from `encoded`). Blend at `blend_strength=1.0` with small mask should show structure from the cut region only.

---

## Section 3: fill_type × mask interaction (~3 min)

All runs `image_purpose=img2img`.

### 3.1 Solid fills (black / white / custom_color)

- [ ] **Expected:** Cut region fills with the chosen solid color. `custom_color` respects `fill_color` hex. No errors.

### 3.2 `noise` and `random`

- [ ] **Expected:** Cut filled with Gaussian noise (`noise`) or uniform random (`random`).

### 3.3 `DazNoise:*` (CUDA-bound fills)

- [ ] **Steps:** Cycle through `DazNoise:pink`, `DazNoise:plasma`, `DazNoise:brown`, `DazNoise:greyscale`, `DazNoise:gaussian`.
- [ ] **Expected:** All composite successfully. No device mismatch errors.
- [ ] **Why specifically:** DazNoise outputs can land on `cuda:0` while mask is CPU — regression guard.

### 3.4 `fill_image` connected (overrides fill_type)

- [ ] **Steps:** Connect a second LoadImage to `fill_image`. Queue.
- [ ] **Expected:** Cut region replaced by the scaled `fill_image` instead of the noise pattern. Composite works across the output_image_mode matrix.

---

## Section 4: Edge cases (~5 min)

### 4.1 Mask connected but no image

- [ ] **Steps:** Disconnect `image`. Keep mask connected. Queue.
- [ ] **Expected:** Node runs without error. Mask is ignored silently; IMAGE = fill pattern.

### 4.2 All-zeros mask (fully hidden)

- [ ] **Steps:** Paint mask fully black (0). Queue.
- [ ] **Expected:** IMAGE output is fully the background fill, no input image pixels visible.

### 4.3 All-ones mask (fully visible)

- [ ] **Steps:** Paint mask fully white (1). Queue.
- [ ] **Expected:** IMAGE output is the full transformed input image, no fill visible.

### 4.4 Partial / feathered mask (smooth gradient)

- [ ] **Steps:** Paint a soft-edge mask.
- [ ] **Expected:** Smooth blending at mask edges — no hard banding, alpha interpolation looks natural.

### 4.5 Mask at different resolution than image

- [ ] **Steps:** Use a mask whose native dimensions differ from the input image (e.g., 512×512 mask with a 1024×1024 image).
- [ ] **Expected:** Mask auto-fits (nearest-exact) to final calculated output dims. No shape-mismatch error.

### 4.6 Batch size > 1

- [ ] **Steps:** Set `batch_size = 4`. Queue.
- [ ] **Expected:** Composite broadcasts across batch. Each of the 4 images shows the cut. No error.

### 4.7 Rapid mask edit iteration

- [ ] **Steps:** Fixed seed. Queue. Edit mask in MaskEditor, save, queue. Repeat 3× with small mask changes.
- [ ] **Expected:** Each queue produces a fresh re-execution reflecting the latest mask (IS_CHANGED working).

### 5.2 Info output omits mask when skipped

- [ ] **Expected:** Preview thumbnail shows composite (cut image), not the uncomposited input.

## What the automated tests DON'T cover

- **Visual correctness** — whether the cut looks right requires a human eye on the preview.
- **ComfyUI cache invalidation through IS_CHANGED** — the JS/Python cache interaction is hard to mock; only a real ComfyUI run proves it works.
- **GPU device placement of DazNoise fills** — mocks use CPU; the cuda:0 path is not exercised in CI.
- **MaskEditor iteration loop** — painting in clipspace and re-queuing is inherently interactive.
- **Interaction with KSampler downstream** — sampler semantics for `latent["noise"]` with `use_as_noise=True` are outside this node's tests.
- **Preview thumbnail composition** — visual.

## Reset / cleanup

Nothing persistent is written by this feature. To reset after testing:

- Close and reopen the workflow, or
- Restart ComfyUI.

No config files are touched. No files are written outside ComfyUI's normal output directory.
